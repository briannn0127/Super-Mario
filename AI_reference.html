Refinement and Explanation

AI-generated suggestions were not copied blindly. The implementation was repeatedly tested in Cocos Creator preview and Firebase Hosting. Several fixes were made after visual testing, including asset cropping, mushroom movement, enemy movement, turtle behavior, big Mario damage, leaderboard loading, cache control, and menu button hitbox alignment.

The main game logic uses simple rectangle-based physics suitable for this assignment: each object has position, size, velocity, and collision state. The player and enemies are moved by velocity and gravity, then corrected against solid blocks. Enemy defeat is only triggered when the player is falling onto the enemy from above; side contact damages the player.

The Firebase leaderboard uses the Realtime Database REST API. Scores are posted as: { name: string, score: number, createdAt: timestamp }. The game loads all leaderboard records, sorts them by score in descending order, and displays the top 10. This avoids requiring a Firebase .indexOn rule for score.
